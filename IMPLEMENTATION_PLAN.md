# VSCode 功能全量复刻实现计划

## 1. VSCode 核心功能模块清单

### 1.1 编辑器核心 (Editor Core)
- **基础编辑**: 语法高亮、智能补全 (IntelliSense)、代码折叠、多光标、Minimap。
- **导航系统**: 面包屑 (Breadcrumbs)、大纲视图 (Outline)、快速打开 (Ctrl+P)。
- **搜索替换**: 文件内查找、跨文件全局搜索 (Ripgrep)、正则支持。

### 1.2 工作区管理 (Workspace)
- **文件资源管理器**: 树状目录、拖拽操作、右键菜单、文件监听。
- **多工作区**: 支持同时打开多个文件夹，工作区配置文件 (.code-workspace)。
- **标签页管理**: 分组显示、预览模式、脏状态标记。

### 1.3 Git 版本控制 (Idea Level)
- **变更管理**: 暂存/取消暂存、差异对比 (Diff)、提交信息编写。
- **历史可视化**: 线性时间线、分支合并图谱 (Graph)、提交详情查看。
- **高级操作**: 分支创建/切换/合并、Rebase、Cherry-pick、冲突解决编辑器。

### 1.4 调试与终端 (Debug & Terminal)
- **集成终端**: 支持多实例、分屏、自定义 Shell (zsh, bash, pwsh)。
- **调试器**: 断点管理、变量监视、调用堆栈、DAP 协议支持。
- **运行配置**: `launch.json` 和 `tasks.json` 解析与执行。

### 1.5 扩展生态 (Extensions)
- **插件市场**: 在线浏览、安装、卸载、启用/禁用。
- **贡献点系统**: 允许插件注入命令、菜单、视图、语言支持、主题。
- **API 支持**: 提供文件系统、窗口、Git、LSP 等后端能力调用。

### 1.6 个性化与设置
- **主题系统**: 颜色主题、文件图标主题、产品图标主题。
- **快捷键**: 默认键位映射、用户自定义键位绑定 (keybindings.json)。
- **配置管理**: 用户设置、工作区设置、语言特定设置。

## 2. 项目概述
本项目旨在基于 **Wails v3** 后端框架，结合 **Vue 3 (Naive UI)** 或 **React 18 (Ant Design/MUI)** 前端技术栈，全量复刻 Visual Studio Code 的核心功能。目标是打造一个跨平台、高性能且具备 Idea 级别 Git 可视化的现代化代码编辑器。

## 3. 核心技术架构

### 3.1 技术选型对比表

| 维度 | Wails v3 + Vue 3 (Naive UI) | Wails v3 + React 18 (Ant Design) |
| :--- | :--- | :--- |
| **开发效率** | ⭐⭐⭐⭐⭐ (SFC 直观，逻辑复用灵活) | ⭐⭐⭐⭐ (JSX 灵活但需更多样板代码) |
| **类型安全** | ⭐⭐⭐⭐ (模板推导稍弱，组合式 API 良好) | ⭐⭐⭐⭐⭐ (TS 原生支持，生态最完善) |
| **性能表现** | ⭐⭐⭐⭐⭐ (Proxy 响应式，更新精准) | ⭐⭐⭐⭐⭐ (Fiber 架构，并发特性强) |
| **组件质量** | ⭐⭐⭐⭐⭐ (Naive UI 主题系统先进) | ⭐⭐⭐⭐⭐ (Ant Design 企业级标准) |
| **学习曲线** | ⭐⭐⭐⭐⭐ (平缓，适合快速上手) | ⭐⭐⭐ (Hooks 心智模型较复杂) |
| **打包体积** | ⭐⭐⭐⭐ (Vue 3 ~33kb, Naive UI 按需) | ⭐⭐⭐ (React ~42kb, Antd 较大) |

### 3.2 通信机制设计
- **同步调用**: 前端直接调用 Go 导出的方法（如 `ReadFile`, `GitStatus`）。
- **异步事件**: 后端通过 `runtime.EventsEmit` 推送文件变更、Git 进度等实时消息。
- **流式传输**: 针对大文件或终端输出，采用分块流式处理以避免内存溢出。

## 4. 模块化实现方案

### 4.1 基础环境与布局系统
- **任务**: 实现 VSCode 风格的五栏布局（活动栏、侧边栏、编辑区、底部面板、状态栏）。
- **Vue 方案**: 使用 `NLayout`, `NLayoutSider` 配合自定义 CSS Grid 实现拖拽调整大小。
- **React 方案**: 使用 `SplitPane` 组件或 MUI 的 `Grid` 系统实现响应式布局。
- **关键点**: 实现窗口标题栏集成（Frameless Window）及多主题切换逻辑。

### 4.2 文件资源管理器 (Explorer)
- **任务**: 实现虚拟文件系统树、文件增删改查、拖拽排序。
- **后端**: 封装 `os` 包实现递归目录读取，增加文件监听 (`fsnotify`)。
- **前端**: 
    - Vue: 使用 `NTree` 组件，支持懒加载和右键菜单 (`NDropdown`)。
    - React: 使用 Ant Design 的 `Tree` 组件，结合 Context API 管理文件状态。

### 4.3 核心编辑器增强 (Monaco Integration)
- **任务**: 多标签页管理、智能补全、代码折叠、Minimap 缩略图。
- **实现**: 
    - 封装 Monaco Editor 为通用组件，通过 Wails 调用后端 LSP 服务获取 IntelliSense 数据。
    - 实现“面包屑”导航 (Breadcrumbs) 显示当前符号路径。

### 4.4 Idea 级别 Git 可视化
- **任务**: 分支图谱 (Graph)、提交历史时间线、三向合并冲突解决。
- **后端**: 深度集成 `libgit2`，解析 `.git` 目录生成节点关系数据。
- **前端**: 
    - 使用 Canvas 或 SVG 绘制复杂的分支合并线条。
    - 实现 Diff 视图：并排对比 (Side-by-side) 与行内对比 (Inline)。

### 4.5 插件生态系统 (Extension Host)
- **任务**: 建立插件市场、插件生命周期管理、贡献点注册。
- **架构**: 
    - 后端使用 Go Plugin 加载 `.so` 文件。
    - 前端提供 SDK，允许插件注入 UI 组件（如侧边栏视图）和命令。

### 4.6 全局搜索与替换
- **任务**: 文件名模糊搜索、内容正则搜索、符号跳转。
- **实现**: 后端使用 `ripgrep` (rg) 进行高性能文本检索，前端展示带高亮的搜索结果列表。

### 4.7 集成终端与调试器
- **任务**: 嵌入式终端 (PTY)、断点调试、变量监视。
- **实现**: 
    - 终端: 集成 `xterm.js`，后端通过 `pty` 库桥接系统 Shell。
    - 调试: 实现 DAP (Debug Adapter Protocol) 客户端，支持启动配置 (`launch.json`)。

## 5. 开发路线图 (Roadmap)

| 阶段 | 周期 | 核心目标 |
| :--- | :--- | :--- |
| **Phase 1: 骨架搭建** | 2 周 | 完成 Wails v3 初始化，实现双前端模板，跑通基础 IPC 通信。 |
| **Phase 2: 编辑核心** | 4 周 | 完成文件树、Monaco 集成、多标签页管理及基础保存功能。 |
| **Phase 3: Git 攻坚** | 6 周 | 实现 libgit2 绑定，完成分支图谱绘制及 Diff 对比视图。 |
| **Phase 4: 高级特性** | 4 周 | 集成 LSP 客户端，实现全局搜索、终端及调试器基础框架。 |
| **Phase 5: 插件与优化** | 4 周 | 完善插件 SDK，进行性能调优（虚拟滚动、增量渲染），发布 Beta 版。 |

## 6. 关键技术难点与对策
- **大文件性能**: 采用分块读取策略，前端 Monaco 开启 `largeFileOptimizations`。
- **跨平台兼容**: 利用 Wails v3 的统一 API 屏蔽 Windows/macOS/Linux 差异。
- **状态同步**: 确保 Go 后端文件变更能实时触发前端 UI 更新（通过 Wails Events）。
- **Git 复杂度**: 使用成熟的 `libgit2` 库处理底层逻辑，前端专注于数据可视化。
